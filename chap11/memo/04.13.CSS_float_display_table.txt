#container02 :: after {contents :""; clear : both; display : table;} /**/
: = 같은 동급
:: = 유사 셀렉터 유사 클래스를 의미한다 따라서 위의 코드는 유사 클래스에 무엇을 정욕하겠다 라는뜻.
외부의 다른태그에 효과를 미치지만 효과를 안나오게 하기 위해서 
div태그 뒤쪽에 요소를 빈컨텐트를 주고 클리어 both로 플롯트효과를 없애주며 테이블은 한행으로 보이게끔 하는것.

이렇게 하면 플롯트 효과 때문에 이미지와 텍스트가 같은 행으로 가지 못했던것이 해소가 된다.

플롯트란 노말플로우가 지배하고있는 영역을 벗어난다.
예를들어 노말플로우가 땅이라한다면 플롯트는 땅에서 뛰어진것.
그 것을 끈어주는것이 clear다.

html5 이전에 주로 사용되던 레이아웃 방식
 - <table></table> 사용
 - 모바일에서는 웹페이지를 보기가 어렵게 한다.

 모바일 시대이후 div태그를사용하여 테이블 효과를 낸다.
 - <div> 태그를 사용한 테이블 레이아웃(display:table)

display : table; = 태그안의 내용들이 테이블처럼 보이게한다.

display : table-row;= tr과 같으며, 한행이고 데이터가 크거나 작거나 높이가 같게한다.

display : table-cell; = td 와 같다.
